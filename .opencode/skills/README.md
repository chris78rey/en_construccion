Se han creado **6 skills** complementarios que materializan el contenido del documento `redefinicion_marketing.md` en una página web cohesiva, personalizada y optimizada para conversión de CTOs y CFOs.

**Ubicación en página:** Centro (después de Authority)
**Personas:** Ambas (CTO por features técnicas, CFO por ROI)
**Integración:** Usa los 4 servicios definidos en redefinición

### 6. **ideal-customer-profile-dattica**
**Propósito:** Definición de 2 ICPs con buyer journey personalizado

**ICP #1: CTO (Technical Decision-Maker)**
- Edad: 35-55 años
- Empresa: Midmarket a Enterprise
- Pain points principales: Deuda técnica, costos OPEX, silos de datos, IA fragmentada
- Buyer journey: 3-6 meses de ciclo de venta
- Messaging: Latencia, agilidad, control
- CTAs preferidas: "Sesión técnica", "Architecture diagram", "Benchmark"

**ICP #2: CFO (Business Decision-Maker)**
- Edad: 40-60 años
- Empresa: Midmarket a Enterprise regulado
- Pain points principales: OPEX descontrolado, riesgo regulatorio, talento escaso, IA limitada
- Buyer journey: 1-2 meses de ciclo de venta
- Messaging: ROI, riesgo, cumplimiento
- CTAs preferidas: "ROI Calculator", "Propuesta CFO", "Llamada peer"

**Persona Detection Logic:**
- Behavioral signals: páginas visitadas (rust, spdp, roi, etc.)
- Dwell time: >10min = CTO, 6-10min = CFO
- localStorage para persistencia
- Google Analytics events para tracking

**Personalization en Página:**
- Mostrar/ocultar secciones por persona
- CTAs personalizados (sesión técnica vs. ROI calc)
- Case studies alternos (métricas técnicas vs. business)
- Email flows diferenciados (post-conversion)

**Ubicación en página:** Aplicado transversalmente
**Dinámicas:** Detector automático al scroll + selector manual
**Integración:** Multi-threading strategy (contactar CTO + CFO + CISO)

  ├─ 325K transacciones/40s
  ├─ 1.2M usuarios simultáneos
  ├─ 20+ años misión crítica
  ├─ 85% reducción auditoría
  └─ Statement de autoridad

SERVICES (skill: services-architecture-dattica)
  ├─ Modernización Legacy (A)
  ├─ Ultra-Baja Latencia (B)
  ├─ IA Soberana (C)
  ├─ Auditoría KNIME (D)
  └─ Bundles (Starter/Growth/Complete)

BLOG TEASER (skill: thought-leadership-content-dattica)
  ├─ Últimos 3 artículos (Rust, SPDP, Oracle)

## Deployment en Coolify

Usar `docker-compose.yml` existente:
- Servicio `web` expone puerto 8080
- Labels Traefik gestionados por Coolify
- Healthcheck: `wget http://localhost:8080/`
- Imagen: nginx:1.26.3-alpine (optimizada)

```bash
docker compose -f docker-compose.yml build
docker compose -f docker-compose.yml up -d
```
